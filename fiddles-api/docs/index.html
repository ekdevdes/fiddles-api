Fiddles

Base URL: http://api.fiddlesapp.co/v2/

Response Format

All API possible responses fall under two categories, they are listed below.

NOTE: The response code of successful requests will always be 0 and the response code of requests whose endpoints do not successfully execute is any number greater than 0. The specific number tells you what kind of error occured.

Success

The JSON payload of the response of all API calls whose endpoints are successfully executed without error will look as follows.

{
  "meta" : {
    "resp_code" : 0,
    "resp_msg" : "", // optional, a short text response from the API endpoint
    "get_params" : {
      "key" : "value",
      ....
    },
    "post_params" : {
      "key" : "value", // Please note, passwords passed in via the HTTP POST method are the only POST parameter that will not be returned
      ....
    }
  },
  "resp" : {
    ... optional additional response information ...
  }
}

Error

The JSON payload of the response of all API calls whose endpoints are not successfully executed without error will look as follows.

{
  "meta" : {
    "resp_code" : 1, // any number greater than 0, the specific number tells the kind of error that occured
    "err_msg" : "", // why the error occured
    "get_params" : {
      "key" : "value",
      ....
    },
    "post_params" : {
      "key" : "value", // Please note, passwords passed in via the HTTP POST method are the only POST parameter that will not be returned
      ....
    }
  },
  "resp" : {
    ... optional additonal response information ....
  }
}

Endpoints

parse

Turns pre-processor code into appropriate HTML, CSS or JS language and creates {{slug}}.html on the server and populates it with the parsed contents of the fiddle. Updates the file on the server if it already exists.

GET PARAMETERS

REQUIRED slug the five character unique identifier of the fiddle to parse

REQUIRED lhtml html language of fiddle to parsePOSSIBLE OPTIONS

  • html
  • haml
  • markdown
  • md

REQUIRED lcss css language of fiddle to parsePOSSIBLE OPTIONS

  • css
  • less
  • sass
  • scss

REQUIRED ljs js language of fiddle to parsePOSSIBLE OPTIONS

  • js
  • javascript
  • cs
  • coffeescript

POST PARAMETERS

REQUIRED html html code of fiddle to parse

REQUIRED css css code of fiddle to parse

REQUIRED js js code of fiddle to parse

REQUIRED opts a JSON payload of other fiddle options

{
  "name" : "iOS 7 Style Menu", // default "Untitled"
  "minify_css" : "true", // default "false"
  "normalize_css" : "true", // default "false"
  "reset_css" : "true", // default "false"
  "prefix_free" : "true", // default "false"
  "library" : "jquery", // default "none". Other options: jquery, prototype, mootools
  "resources" : { // by default no resources are included
    "js" : [ // empty by default
      "http://example.com/source.js",
      "http://code.jquery.com/jquery-latest.js"
    ],
    "css" : [ // empty by default
      "http://example.com/lib.css",
      "http://twitter.com/bootstrap/latest.css"
    ]
  }
}

ERROR CODES

1 translation not supported
2 syntax error
3 parsing error NOTE: Can be combined to describe multiple parsing errors, see below
   3.1 haml parsing error
   3.2 less parsing error
   3.3 sass/scss parsing error
   3.4 coffeescript parsing error

EXAMPLE RESPONSE

SUCCESS

{
  ... meta ...
  "resp" : {
    "url" : "http://ek.alphaschildren.org/fiddles/abc123.html",
    "result" : {
      "html" : "... code ...",
      "css" : "... code ....",
      "js" : "... code ...",
      "full_html" : ".... contents of {{slug}}.html file ..."
    }
  }
}

ERROR

{
  "meta" : {
    "resp_code" : "3.1, 3.2", // 1 or 2 or 3.1, 3.2, 3.3, 3.4
    "err_msg" : "haml parsing error, less parsing error", // why the error occured
    "get_params" : {
      "key" : "value",
      ....
    },
    "post_params" : {
      "key" : "value", // Please note, passwords passed in via the HTTP POST method are the only POST parameter that will not be returned
      ....
    }
  },
  "resp" : {
    "url" : "http://ek.alphaschildren.org/fiddles/abc123.html",
    "haml" : {
      "errs" : {
        "Unexpected '%' on line 5",
        "Unknown Character '<' on line 95"
      },
      "count" : 2
    },
    "less" : {
      "errs" : {
        "Unexpected '{' on line 5",
      },
      "count" : 1
    }
  }
}
